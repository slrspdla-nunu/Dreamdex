# 🌙 Dreamdex — Dream Archive

> "꿈은 사라지는 기록이 아니라, 수집 가능한 데이터이다."

꿈을 기록하면 꿈 속 **장소·인물·상황**을 자동으로 분류·수집해주는 **디지털 무의식 보관소**.
단순한 꿈 일기(CRUD 메모장)가 아니라, 기록이 쌓일수록 자신의 무의식 세계를 **수집하고 탐험**하는 아카이브 서비스입니다.

## ✨ 주요 기능

| 기능 | 설명 |
|---|---|
| **꿈 기록** | 제목·날짜·내용·감정(6종) 기록. 저장 즉시 키워드 자동 수집 피드백 |
| **대시보드(홈)** | 피처드 꿈 · 수집 진행 · 감정 흐름 미니차트 |
| **아카이브** | 카드 목록 · 검색 · 감정 필터 · 즐겨찾기 |
| **Dreamdex 도감** | 꿈 내용에서 키워드를 자동 분류해 도감에 등록 · 발견/미발견 슬롯 · 완성률 |
| **통계** | 총 꿈 수 · 최다 감정/장소 · 최근 30일 · 감정 분포 도넛 · 감정 흐름 |
| **꿈 지도** | 같은 꿈에 함께 등장한 요소를 연결한 동시출현 그래프 |
| **수집 시스템** | 꿈 10/30/50개 마일스톤 해금(지도 개방 · 희귀 키워드 · 숨겨진 도감) |
| **설정** | 테마 3종(천문대/필사본/수장고) · 예시 데이터 · JSON 백업/복원 · 초기화 |

## 🛠 기술 스택

- **Vanilla HTML / CSS / JavaScript** (프레임워크·빌드 도구 없음)
- **LocalStorage** 단일 데이터 저장소 (서버·로그인 없음)
- **Chart.js** (CDN, 통계 도넛 차트 — 없으면 폴백 막대그래프)
- 해시 기반 SPA 라우팅 → 정적 호스팅 / 더블클릭 실행 가능

## 🧠 핵심 설계: 키워드 자동 분류

오프라인/LocalStorage 환경이므로 AI 호출 없이 **사전(dictionary) 기반 규칙 매칭**으로 구현했습니다.

- 각 키워드 = `표제어 + 동의어 배열` (예: `바다 = [바다, 해변, 파도, 바닷가]`)
- 꿈의 제목+내용에서 동의어가 매칭되면 "발견"으로 등록
- 사전 규모 = 도감 분모: **장소 50 / 인물 30 / 상황 40**
- 결정적(deterministic)이라 테스트가 쉽고, 도감 완성률이 명확

```
dreams[]  ─┬─→ classify()  → 도감(Dreamdex)
           ├─→ buildStats() → 통계
           └─→ buildGraph() → 꿈 지도
```
> 도감·통계·지도는 저장하지 않고 `dreams`에서 **매번 파생 계산**한다.

## 📁 구조

```
꿈일기/
├─ index.html
├─ css/styles.css
├─ js/
│  ├─ dictionary.js   # 키워드 사전 (장소50/인물30/상황40, tier: common/rare/hidden)
│  ├─ store.js        # LocalStorage 데이터 계층 + 감정/테마 정의
│  ├─ classify.js     # 분류 엔진 + 도감/통계/지도/해금 파생
│  ├─ sampledata.js   # 예시(데모) 꿈 데이터
│  ├─ icons.js        # 커스텀 SVG 글리프(감정·식물·지역·로고·UI)
│  ├─ charts.js       # 감정 차트(Chart.js) + 꿈 지도(Canvas force layout)
│  ├─ views.js        # 화면 렌더링(정원·도감·식물상세 등) + 공용 UI
│  └─ app.js          # 해시 라우터 + 부팅
└─ docs/
   ├─ 01_service-plan.md            # 1단계 서비스 기획서
   └─ 02_information-architecture.md # 2단계 IA
```
