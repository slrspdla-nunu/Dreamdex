# Dreamdex : Garden of Dreams — 서비스 기획서 (개정판)

> "꿈은 씨앗이다."

## 1. 서비스 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | **Dreamdex : Garden of Dreams** (약칭 Dream Garden) |
| 한 줄 정의 | 꿈을 기록하면 꿈 속 요소가 **씨앗**이 되어, 나만의 **무의식 정원**에서 자라나는 꿈 아카이브 |
| 형태 | 클라이언트 사이드 웹앱 (HTML / CSS / JavaScript) |
| 저장 | LocalStorage (서버·로그인 없음, 단일 사용자) |
| 결합 구조 | 꿈 아카이브 + 꿈 도감 + **수집** + **성장(정원)** |

## 2. 컨셉 전환 — 무엇이 바뀌었나

기존(Dreamdex)은 꿈 속 요소를 **수집(breadth)** 하는 도감 서비스였다.
개정판은 여기에 **성장(depth)** 축을 더한다.

> 같은 요소를 **반복해서 꿈꿀수록** 그 씨앗이 자라 식물이 되고, 정원이 풍성해진다.

- **수집(꿈 도감)** = 얼마나 *다양한* 요소를 만났는가 (넓이)
- **성장(정원/식물 도감)** = 얼마나 *깊이* 꿈꿨는가 (깊이)

두 축이 서로를 보완해, "한 번 더 기록할 이유"를 만든다.

## 3. 핵심 루프

```
꿈 기록 → 키워드 추출(자동) → 씨앗 생성 → 정원에 심김
   ↑                                         ↓
   └────────── 반복 기록 ←── 성장(씨앗→새싹→꽃/나무→만개) ──┘
```

예시) "학교에서 친구와 시험을 봤다."
→ 추출: `학교 · 친구 · 시험`
→ 생성: **학교의 씨앗 · 친구의 씨앗 · 시험의 씨앗**
→ 다음에 또 `학교`를 꿈꾸면 → *학교의 씨앗*이 **새싹**으로 성장

## 4. 식물 모델 (키워드 = 식물, 감정 = 외형)

새 저장 데이터 없이 **`dreams[]`에서 전부 파생**한다. 각 키워드 1종 = 식물 1종.

### (1) 성장 단계 — 키워드 등장 횟수로 결정
| 단계 | 조건(누적 등장) | 의미 |
|---|---|---|
| 🌱 씨앗 | 1회 | 정원에 막 심김 (식물 도감 **미등록**) |
| 🌿 새싹 | 2회 | **식물 도감 "발견" 등록** (재배 성공) |
| 🪴 자란 식물 | 3–4회 | 형태가 드러남 |
| 🌸 만개 | 5–7회 | 꽃/나무가 활짝 |
| ✦ 고목/광휘 | 8회+ | 최고 성장 (정원의 랜드마크) |

> **두 도감의 분기점:** 꿈 도감은 1회(관찰)부터, 식물 도감은 2회(새싹=재배)부터 등록.

### (2) 외형 = 카테고리(형태) × 감정(색·분위기)
- **형태(카테고리)**: 장소 → 나무/큰 식물 · 인물 → 꽃 · 상황 → 덩굴/풀
- **색·분위기(대표 감정)**: 그 키워드가 등장한 꿈들의 **최빈 감정**으로 결정

| 감정 | 식물 외형 |
|---|---|
| 행복 | 밝은 금빛 꽃 |
| 신기함 | 빛나는 청록 꽃 |
| 불안 | 호박빛 가시 덩굴 |
| 공포 | 검은 장미 |
| 슬픔 | 푸른 (우는) 꽃 |
| 그리움 | 보랏빛 꽃 |

→ 형태 3 × 감정 6 = 18계열 + 성장 5단계의 시각 변주로 정원이 다채로워진다.

### (3) 희귀도(종 등급) — 사전 tier 기반
- 일반 식물 = common 키워드 (약 92종)
- **희귀 식물** = rare 키워드 (약 19종) — 꿈 30개부터 발견 가능
- **전설 식물** = hidden 키워드 (약 9종) — 꿈 50개부터 발견 가능

> 성장 단계(얼마나 키웠나)와 희귀도(종의 등급)는 **별개 축**. 흔한 식물도 만개시킬 수 있고, 희귀 식물도 씨앗 단계일 수 있다.

## 5. 지역(Region) 해금 — 정원이 넓어진다

꿈 기록 수에 따라 새 **지역(생물군계)** 이 열리고, 식물이 그곳으로 이주·전시된다.

| 마일스톤 | 해금 지역 | 들어오는 식물 |
|---|---|---|
| 시작 | **씨앗밭(묘목장)** | 모든 새 씨앗의 첫 거처 |
| 꿈 10개 | **달빛 온실** | 인물 식물(꽃) |
| 꿈 30개 | **기억의 숲** + 희귀 식물 개방 | 장소 식물(나무) |
| 꿈 50개 | **잊힌 도서관** + 전설 식물 개방 | 상황 식물(덩굴) |
| 꿈 100개 | **심연의 정원** | 희귀·전설·고목(만개) 전시 |

> 기존 10/30/50 수집 해금을 흡수·확장(+100). 지역 개방은 정원이 시각적으로 풍성해지는 보상.

## 6. 기능 범위

### 유지 (기존)
꿈 기록 · 꿈 목록(아카이브) · 꿈 상세 · 꿈 도감 · 통계 · 꿈 지도

### 추가 (이번 개편)
1. **정원(Garden)** — 홈이자 핵심 경험. 지역별로 성장한 식물 배치.
2. **성장 시스템** — 반복 등장 키워드의 단계적 성장 + 성장 피드백.
3. **감정 기반 식물** — 대표 감정으로 외형 결정.
4. **식물 도감** — 꿈 도감과 별개의 재배 컬렉션(일반/희귀/전설).
5. **식물 상세** — 성장 단계·대표 감정·희귀도·등장한 꿈·소속 지역.
6. **지역 해금** — 10/30/50/100 마일스톤.

## 7. UX 원칙

- "작성"보다 **"수집·성장"**의 재미 — 기록 직후 *씨앗 생성/성장* 피드백이 보상.
- **게임화는 있되 게임이 아니다** — 점수·랭킹 없음. "정원이 채워진다"는 정적·아카이브적 보상.
- 정원은 **조용히 살아있는 화면** — 부드러운 성장 애니메이션, 별빛·달빛 톤.

## 8. 디자인 키워드
몽환적 · 신비로움 · 디지털 아카이브 · 별빛 · 달빛 · 식물 · 기억 · 무의식 · 꿈
(기존 테마 3종 — 천문대/필사본/수장고 — 위에 정원 비주얼을 얹는다. 식물은 전부 커스텀 SVG, 이모지 미사용.)

## 9. 성공 지표 (포트폴리오 관점)
- 기획→IA→플로우→데이터→구현의 단계적 산출물.
- **꿈 1개 기록 → 씨앗 → (반복) → 식물 → 정원**의 핵심 경험이 실제로 동작·시연.
- 빈 상태(묘목장)부터 풍성한 정원까지, 예시 데이터로 전 기능 시연 가능.
